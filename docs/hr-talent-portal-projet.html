🎯 1. Vision du Projet

Le HR Talent Portal est une plateforme de recrutement end-to-end développée sur Salesforce pour Talan Tunisie. Elle automatise et enrichit l'ensemble du processus de recrutement grâce à l'intelligence artificielle :

🤖 Scoring IA automatique

Chaque CV déposé est analysé par l'IA (LLaMA 3.3 70B) qui attribue un score de compatibilité, une recommandation et identifie les points forts/faibles

🎮 Test Gaming gamifié + IA

Évaluation ludique avec questions générées dynamiquement par l'IA (Groq LLaMA 3.3 70B) adaptées au poste, timer par question, bonus de vitesse, validation serveur (réponses jamais côté client) et système anti-triche

📊 Dashboard Recruteur

Vue pipeline complète avec KPIs, filtres avancés, planification (individuelle + lot), gestion des offres et génération IA de descriptions

🌐 Portail Candidat

Espace personnel avec suivi des candidatures en temps réel, profil éditable, carte interactive des postes, et accès aux tests gaming

📋 2b. Analyse des Besoins

Cette section identifie les acteurs impliqués dans le système, tout en détaillant les exigences fonctionnelles et non fonctionnelles.

Exigences Fonctionnelles

Pour les visiteurs (Utilisateurs non-identifiés)

  • Consulter les offres d'emploi : Accéder au catalogue des postes vacants publiés sur le portail public.
  • Déposer une candidature : Soumettre un formulaire de postulation en téléversant un curriculum vitae (Upload CV).

Pour les candidats (Après authentification sur l'espace B2C)

  • S'authentifier : Accéder de manière sécurisée à son espace personnel.
  • Suivre l'avancement du dossier : Visualiser en temps réel le statut de ses candidatures (en cours, validé, refusé).
  • Gérer son profil B2C : Mettre à jour ses informations personnelles et ses compétences techniques.
  • Passer le Test Gaming : Réaliser les tests ludiques d'évaluation des soft skills intégrés à la plateforme.

Pour les recruteurs (Back-office Salesforce)

  • Publier les offres d'emploi : Créer et diffuser de nouveaux besoins en recrutement sur Experience Cloud.
  • Orchestrer l'IA (Parsing NLP) : Lancer l'analyse automatique des documents déposés.
  • Générer des questions Gaming par IA : Déclencher la génération automatique de questions adaptées au poste via Groq LLaMA 3.3 70B.
  • Planifier les entretiens : Coordonner les rendez-vous entre les candidats retenus et les experts techniques.
  • Envoyer des notifications : Automatiser la communication par e-mail avec les candidats à chaque étape (Staging).

Pour les managers (Experts évaluateurs)

  • Consulter les profils candidats : Accéder aux dossiers qualifiés par le moteur de matching IA.
  • Saisir le feedback évaluateur : Consigner les comptes-rendus d'entretiens et les notes techniques.
  • Valider ou refuser un candidat : Prendre une décision de recrutement basée sur les résultats des tests et le score de matching IA.

Pour le responsable RH (Gouvernance et Pilotage)

  • Consulter les Dashboards & KPIs : Piloter la performance globale du recrutement (Time-to-Hire, qualité du sourcing).
  • Gérer l'Onboarding : Déclencher les processus d'intégration pour les candidats ayant accepté l'offre.
  • Générer les contrats : Automatiser la création et le suivi des documents contractuels d'embauche.

Pour le système IA (Acteur système automatisé)

  • Analyse NLP : Extraire les données sémantiques des fichiers PDF/Docx.
  • Calcul du Match Score : Générer un score de compatibilité entre le profil du candidat et les exigences du poste.
  • Génération de questions Gaming : Créer 25 questions dynamiques adaptées au poste via l'API Groq.

Exigences Non Fonctionnelles

🎨 Convivialité & Ergonomie

Interfaces intuitives et faciles à utiliser, design responsive et moderne

⏰ Disponibilité

L'application doit être toujours disponible et accessible aux utilisateurs (hébergée sur Salesforce Cloud)

🔒 Sécurité

Profils avec autorisations spécifiques, token SHA-256, rate limiting, audit logging, FLS enforced

Architecture modulaire permettant l'ajout de caractéristiques futures et la maintenance continue

Flux détaillé :

  1. Le candidat parcourt les offres sur le portail Experience Cloud et postule via le formulaire multi-étapes
  2. Un Lead est créé automatiquement avec les infos du candidat
  3. Le CV est uploadé (upload chunké jusqu'à 10 Mo) → le trigger CVScoringTrigger se déclenche
  4. L'IA analyse le CV via Groq API : extraction du texte, comparaison avec le poste, scoring (Tech 40%, Expérience 25%, Formation 20%, Soft Skills 15%)
  5. Score > 15% → Lead qualifié → Flow auto-conversion : Lead → Contact + Account + Opportunity (stage « Gaming »)
  6. Un utilisateur portail est créé automatiquement + email de bienvenue avec identifiants
  7. Le recruteur planifie le test gaming (individuel ou en lot) → email d'invitation envoyé
  8. Le candidat passe le test (10 questions adaptées au poste, timer, bonus vitesse)
  9. Score ≥ 60% → passage automatique à « Technique » | Score < 40% → refusé
  10. Le recruteur avance le candidat : Technique → Architecture → RH Fit → Décision finale
  11. Email de notification envoyé au candidat à chaque changement d'étape

🎮 6. Test Gaming Gamifié

Un système d'évaluation ludique et engageant avec génération dynamique de questions par IA (Groq LLaMA 3.3 70B), remplaçant les questions statiques hardcodées.

🤖 Questions générées par IA

25 questions adaptées au poste générées via Groq LLaMA 3.3 70B. Le recruteur clique sur « Générer questions Gaming » depuis le dashboard. Fallback statique (4 banques) si aucune question IA.

🧩 3 Types de questions

QCM classique (~15), Puzzle logique/mathématique (~5), Ordonnancement (~5) — répartition automatique par l'IA

⏱️ Timer dynamique par difficulté

Facile: 30s, Moyen: 25s, Difficile: 20s, Ordonnancement: 35s — auto-avance au timeout

⚡ Validation serveur sécurisée

Les réponses correctes ne sont jamais envoyées au client. Chaque réponse est validée côté serveur via validateAnswer(). +2 bonus vitesse si <50% du temps.

🔄 Architecture dynamique (Avant / Après)

AspectAvant (statique)Après (dynamique IA)
Source des questions100 questions hardcodées dans le JS25 questions générées par IA par offre, stockées dans Gaming_Question__c
Réponses correctesVisibles dans le code source clientJamais envoyées au client — validation serveur uniquement
Adaptation au posteDétection basique par catégorieQuestions 100% adaptées au titre et description du poste
Sélection10 aléatoires parmi 25 (client)10 aléatoires via Crypto.getRandomInteger() (serveur)

Déroulement du test

  1. Le recruteur génère les questions IA depuis le dashboard (🤖 bouton par offre)
  2. GamingQuestionService (Queueable) appelle Groq API → 25 questions stockées en base
  3. Le candidat reçoit l'email d'invitation avec le lien du test
  4. 10 questions aléatoires sont servies par Apex (sans réponses correctes)
  5. Compte à rebours 3-2-1 puis le quiz démarre
  6. Timer individuel par question avec barre de progression visuelle
  7. Chaque réponse est validée côté serveur en temps réel
  8. Révélation animée du score final
  9. Score ≥ 60% → passage automatique à l'étape « Technique »
  10. Score < 40% → candidature automatiquement refusée (Closed Lost)

🔒 Système Anti-Triche — Mode Sécurisé

Le test intègre un système de surveillance en temps réel qui détecte toute tentative de quitter la fenêtre du test :

  • Détection multi-événements : visibilitychange (changement d'onglet) + blur (Alt+Tab, clic hors fenêtre, notification)
  • Avertissement préalable : sur l'écran d'accueil, un bandeau rouge informe le candidat des règles
  • 1ère violation → Overlay d'avertissement rouge avec animation shake : « Au prochain changement, le test sera terminé avec 0% »
  • 2ème violation → Test automatiquement terminé, score 0% soumis à Salesforce, écran « Violation détectée » avec bouclier rouge
  • Les listeners sont activés au début du quiz et nettoyés proprement en fin de test (pas de fuites mémoire)

Technologies : document.visibilitychange, window.blur, soumission Apex automatique via submitTestResults

🔐 12. Authentification & Sécurité

Le portail implémente une architecture de sécurité multi-couches conforme aux bonnes pratiques OWASP, comprenant 2 mécanismes d'authentification, un système de réinitialisation par token sécurisé, et un journal d'audit complet.

12.1 — Mécanismes d'Authentification (2 modes)

🔑 Login classique (Email)

Authentification via Site.login() avec résolution automatique email → username. Le candidat saisit son email, le système retrouve le User Salesforce correspondant et authentifie via le username interne.

️ Profil Experience Cloud

Customer Community Login User — accès limité au portail via permissions contrôlées. Résolution dynamique du ProfileId (pas de hardcoding).

🌐 Routage Community dynamique

Tous les liens internes (login, reset, accueil) utilisent un getter communityBasePath qui détecte dynamiquement le préfixe /home2 depuis window.location.pathname — aucun hardcoding d'URL.

12.2 — Système de Réinitialisation Sécurisée par Token

🔒 Architecture Zero-Knowledge — Aucun mot de passe dans les emails

Contrairement à l'approche classique (envoi d'un mot de passe temporaire en clair), le portail implémente un système token-based inspiré des standards OAuth/PKCE :

  1. Génération du token — Un token aléatoire de 256 bits est généré via Crypto.getRandomInteger() (CSPRNG Salesforce)
  2. Hachage SHA-256 — Seul le hash du token est stocké en base (Password_Reset_Token__c.Token_Hash__c). Le token en clair n'existe qu'en mémoire durant la génération
  3. Email sécurisé — L'email contient uniquement un lien avec le token vers la page /s/set-password. Aucun mot de passe n'est transmis
  4. Validation côté serveur — Le token soumis est re-haché (SHA-256) et comparé au hash stocké. Cela empêche toute exploitation même en cas de fuite de la base
  5. Changement Salesforce natif — Le mot de passe est défini via System.setPassword() après validation du token

Protections anti-attaque

ProtectionMécanismeDétail
ExpirationTTL 24 heuresChaque token expire automatiquement après 24h (Expiration__c)
Usage uniqueFlag Is_Used__cLe token est marqué comme utilisé dès la première validation réussie — impossible de le réutiliser
Rate LimitingMax 3 tentativesCompteur Attempts__c incrémenté à chaque essai. Au-delà de 3, le token est invalidé définitivement
Validation mot de passeRegex case-sensitiveMinimum 8 caractères, 1 majuscule ([A-Z]), 1 minuscule ([a-z]), 1 chiffre ([0-9]), 1 spécial — validé via Pattern.matches()
Messages génériquesAnti-énumérationLe même message d'erreur est affiché que le token soit invalide, expiré ou déjà utilisé — empêche l'énumération
Audit TrailSecurity_Audit_Log__cChaque action (génération, succès, échec, tentative brute-force) est loguée avec timestamp, email, sévérité et détails

12.4 — Composant LWC setPassword

Page de définition de mot de passe sécurisée

  • Extraction du token depuis les paramètres d'URL (currentPageReference)
  • Validation en temps réel : jauges visuelles pour chaque critère (longueur, majuscule, minuscule, chiffre, spécial)
  • Confirmation de correspondance entre les deux champs mot de passe
  • Messages d'erreur en français : traduction des erreurs Salesforce (« mot de passe déjà utilisé », « token expiré », etc.)
  • Écran de succès avec lien de connexion dynamique (basePath auto-détecté)
  • Design responsive avec animations CSS (transitions, hover states, gradient backgrounds)
